# LMSA Firmware Downloader

A standalone tool that downloads firmware files from `rsddownload-secure.lenovo.com`
and `download.lenovo.com` using a **fresh GUID + JWT** captured from a running LMSA session.

When the crawler finishes, its JWT session has expired. This tool lets you download
the files again with new credentials — no separate software required beyond Python 3.9+
and the `requests` library.

---

## How it works (technical)

Files are served via two mechanisms discovered by live testing (2026-03-04):

| URL type | Example | How to download |
|---|---|---|
| **Pre-signed S3** | `rsddownload-secure.lenovo.com/file.zip?X-Amz-Algorithm=…` | `GET` directly — credentials in URL, no auth headers |
| **Plain S3** (from getRomList) | `rsddownload-secure.lenovo.com/file.zip` | Tool fetches fresh pre-signed URL via `getResource.jhtml`, then `GET` |
| **Public CDN** | `download.lenovo.com/lsa/…zip` | `GET` directly — no auth needed |

> ⚠️ **CloudFront blocks HEAD for pre-signed URLs** (returns HTTP 403) even when the URL
> is valid. The tool uses **GET** for all S3 downloads — never HEAD.

The tool exchanges your WUST for a fresh JWT and uses it to call `getResource.jhtml`,
which returns a 7-day pre-signed S3 URL.

## Quick start

### Step 1 — Capture GUID + JWT + WUST via HTTPToolkit

1. Install [HTTPToolkit](https://httptoolkit.com) (free).
2. Click **Intercept → Software Fix (LMSA)** to start a proxied session.
3. Open any device page in LMSA to trigger API calls.
4. In HTTPToolkit, click any `POST lsa.lenovo.com/Interface/…` request:
   - **Request headers** tab → copy `guid` → paste as `guid` in `config.ini`
   - **Request headers** tab → copy `Authorization` (drop `Bearer ` prefix) → paste as `jwt`
5. Find a `POST lsa.lenovo.com/Interface/user/lenovoIdLogin.jhtml` request:
   - **Request body** tab → copy `dparams.wust` → paste as `wust`
   - The WUST enables automatic JWT refresh and is good for ~12 hours.

### Step 2 — Edit `config.ini`

```ini
[auth]
guid = 3adf1304-8a70-4352-b687-8eddcef6b7d1   ; from HTTPToolkit
jwt  = dcwyMSwoE07AHkME1EQt…                  ; from HTTPToolkit (no "Bearer ")
wust = ZAgAAAAAAAGE9MTM…                       ; for auto JWT-refresh

[urls]
urls_file = download_links.txt    ; generated by crawler

[settings]
output_dir = downloads
workers    = 4
```

### Step 3 — Run

```bash
pip install requests              # one-time setup
python downloader.py              # download all files
python downloader.py --dry-run   # inspect URLs without downloading
```

## Supported URL manifest files

| File | Contents |
|---|---|
| `download_links.txt` | Curl commands with session headers — **recommended** |
| `lmsa_firmware_urls.txt` | Tab-separated URL + filename (all files) |
| `lmsa_rom_urls.txt` | ROM / OTA firmware only |
| `lmsa_tool_urls.txt` | Flash-tool archives only |
| `lmsa_plugin_urls.txt` | PC-side LMSA plugins |

You can also add individual URLs inline:

```ini
[urls]
url_1 = https://rsddownload-secure.lenovo.com/fastboot_lamuc_g_…zip
url_2 = https://download.lenovo.com/lsa/Rescue/…zip
```

## Configuration reference

| Section | Key | Default | Description |
|---|---|---|---|
| `[auth]` | `lmsa_jwt` | — | Combined `GUID:JWT` format (alternative to guid + jwt) |
| `[auth]` | `guid` | — | Device GUID from LMSA session |
| `[auth]` | `jwt` | — | Bearer token without `Bearer ` prefix |
| `[auth]` | `wust` | — | WUST for automatic JWT refresh (optional but recommended) |
| `[auth]` | `base_url` | `https://lsa.lenovo.com/Interface` | LMSA API base URL |
| `[auth]` | `verify_ssl` | `true` | TLS certificate verification |
| `[settings]` | `output_dir` | `downloads` | Output directory (relative to config file) |
| `[settings]` | `workers` | `4` | Parallel download threads |
| `[settings]` | `retries` | `3` | Retry count per file |
| `[settings]` | `resume` | `true` | Resume partial downloads |
| `[urls]` | `urls_file` | — | Path to manifest file |
| `[urls]` | `url_N` | — | Inline URL (`url_1 = …`, `url_2 = …`, …) |

## JWT expiry

The LMSA JWT rotates on every API call and the session expires when LMSA closes.

- **With WUST**: the tool automatically refreshes the JWT by calling
  `lenovoIdLogin.jhtml`. The WUST is valid for ~12 hours after your last LMSA login.
- **Without WUST**: capture a new `guid` + `jwt` from HTTPToolkit and update `config.ini`.

## Limitations

- **Demo / elabel firmware** (e.g., `*_elabel_*_demogb.zip`): these files are tied to
  specific hardware devices (demo units) and cannot be downloaded through the standard
  LMSA API without a physically connected device that identifies itself as a demo unit.
  The `getResource.jhtml` endpoint does not return presigned URLs for restricted firmware.

- **Files with expired pre-signed URLs** from the crawler output: the tool automatically
  fetches fresh presigned URLs via `getResource.jhtml` using your current session.
